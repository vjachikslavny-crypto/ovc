# Производительность загрузки PDF

## Оценка времени загрузки

Для PDF файла **7 МБ, 4 страницы**:

### Первый просмотр (без кэша на сервере):

1. **Рендеринг страницы на сервере (PyMuPDF):**
   - Чтение PDF файла: ~10-50ms
   - Рендеринг страницы в изображение: ~100-300ms на страницу
   - Конвертация в WEBP (качество 85%): ~50-100ms
   - Сохранение в кэш: ~10-20ms
   - **Итого на страницу: ~200-500ms**

2. **Передача по сети:**
   - Размер WEBP изображения: ~200-500 КБ на страницу (зависит от содержимого)
   - При локальном сервере (localhost): ~10-50ms на страницу
   - При удаленном сервере: зависит от скорости сети

3. **Декодирование в браузере:**
   - Декодирование WEBP: ~50-100ms на страницу

**Общее время первого просмотра:**
- **Параллельная загрузка (текущая оптимизация):** ~1-2 секунды для всех 4 страниц
- **Последовательная загрузка:** ~2-4 секунды

### Повторный просмотр (с кэшем на сервере):

1. **Чтение из кэша:** ~1-5ms
2. **Передача по сети:** ~10-50ms на страницу (локально)
3. **Декодирование в браузере:** ~50-100ms на страницу

**Общее время повторного просмотра:**
- **~0.5-1 секунда** для всех 4 страниц

## Факторы, влияющие на скорость:

1. **Размер PDF файла:** Больше файл → дольше чтение и рендеринг
2. **Сложность страниц:** Много графики/текста → больше размер WEBP
3. **Масштаб (zoom):** Больше масштаб → больше размер изображения → дольше передача
4. **Наличие кэша:** Кэшированные страницы загружаются в 10-20 раз быстрее
5. **Скорость сети:** Локальный сервер быстрее удаленного
6. **Производительность сервера:** Быстрее CPU → быстрее рендеринг

## Оптимизации в текущей реализации:

1. **Кэширование:** Все отрендеренные страницы сохраняются в `data/uploads/pages/{file_id}/`
2. **Параллельная загрузка:** Для файлов ≤5 страниц страницы загружаются параллельно
3. **Постепенная загрузка:** Для больших файлов страницы загружаются с задержкой
4. **Lazy loading:** Невидимые страницы загружаются через IntersectionObserver
5. **WEBP формат:** Современный формат с хорошим сжатием (85% качество)

## Рекомендации:

- **Для файлов до 10 МБ:** Загрузка всех страниц сразу (~1-2 сек первый раз, ~0.5 сек повторно)
- **Для больших файлов:** Постепенная загрузка при прокрутке
- **Первый просмотр:** Может занять 1-3 секунды (рендеринг на сервере)
- **Повторный просмотр:** Обычно <1 секунды (из кэша)

## Мониторинг производительности:

В консоли браузера (F12) можно увидеть логи:
- `PDF viewer: loading page` - начало загрузки страницы
- `PDF viewer: page loaded successfully` - страница загружена (с временем загрузки)
- `PDF viewer: all X pages loaded in Xs` - все страницы загружены

## Улучшения в будущем:

1. **Прогресс-бар:** Показывать прогресс загрузки страниц
2. **Предзагрузка:** Загружать следующую страницу заранее
3. **Адаптивное качество:** Меньшее качество для предпросмотра, большее при zoom
4. **Сжатие:** Настройка качества WEBP в зависимости от размера файла
5. **Потоковая загрузка:** Загрузка страниц по мере прокрутки (уже реализовано через IntersectionObserver)
